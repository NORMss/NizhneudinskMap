# Карта мест Нижнеудинска

Веб-приложение для отображения мест на карте Нижнеудинска с карточками, категориями и защищенным добавлением новых точек.

## Что реализовано

- Карта (Leaflet + CARTO basemap) с метками по Нижнеудинску.
- Карточка при наведении на метку и popup с быстрым переходом на страницу места.
- Отдельная страница места: название, описание, адрес, координаты, категория, сайт, фото.
- Главная страница в приоритете показывает карту и список мест.
- Добавление места открывается в диалоге после клика по карте или по кнопке "Добавить место".
- Добавление возможно только по паролю (`password` поле в форме).
- Категории:
  - выбор существующей категории;
  - создание новой категории с цветом;
  - цвет категории определяет цвет пина на карте;
  - список категорий сохраняется между перезапусками.
- Фото: загрузка файла до 1 МБ или ссылка.
- Адрес заполняется автоматически по координатам через Nominatim (OpenStreetMap).
- Гео-ограничение: можно добавить точки только в пределах Нижнеудинска.

## Быстрый старт

1. Установить зависимости:

```bash
npm install
```

2. Запустить сервер:

```bash
npm start
```

3. Открыть сайт:

```text
http://localhost:3000
```

## Пароль на добавление

По умолчанию пароль: `nizhneudinsk`

Для продакшена задайте свой пароль через переменную окружения:

```bash
ADD_PLACE_PASSWORD="ваш_сложный_пароль" npm start
```

## Хранение данных

- Точки: `data/places.geojson` (GeoJSON FeatureCollection).
- Категории: `data/categories.json`.
- Загруженные изображения: `uploads/`.

Для места сохраняются поля:

- `name`, `description`, `siteUrl`, `address`, `photo`, `createdAt`;
- `categoryId`, `categoryName`, `categoryColor`.

## API

- `GET /api/places` — список мест (с категорией и цветом).
- `GET /api/places/:id` — одно место.
- `GET /api/categories` — список категорий.
- `POST /api/places` — создание места (требует `password`).

## Замечания

- Поддерживается только загрузка изображений.
- Размер загружаемого фото ограничен 1 МБ.
